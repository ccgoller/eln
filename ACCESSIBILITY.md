# Accessibility Guidelines for This Repository

This document defines accessibility expectations for all documentation and templates in this repository.

## Documentation standards

- Use one `H1` heading per page.
- Do not skip heading levels (`##` should follow `#`, etc.).
- Use concise, descriptive section titles.
- Keep sentences and paragraphs reasonably short.

## Links

- Use descriptive link text that makes sense out of context.
  - Good: “Read the Instructor Setup Guide”
  - Avoid: “Click here”
- Prefer HTTPS links.

## Images and diagrams

- Every image must include alt text that describes its purpose.
- If an image contains critical instructional content, provide the same content in nearby text.
- Avoid screenshots when text instructions are sufficient.

## Tables

- Use a header row in every table.
- Keep cells concise and scannable.
- Do not use tables only for visual layout.

## Color and emphasis

- Do not rely on color alone for meaning.
- Use text labels in addition to color indicators.
- Use bold text sparingly for emphasis, not as structure.

## Code and commands

- Use fenced code blocks for commands and examples.
- Keep command examples copy-paste ready.
- Explain the expected outcome after commands when helpful.

## File naming and structure

- Use consistent naming (for example, `01-...`, `02-...` for ordered tutorials).
- Keep related material grouped in clear folders.
- Prefer predictable paths to reduce navigation overhead.

## Review checklist (before merge)

- [ ] Heading hierarchy is valid.
- [ ] Links are descriptive.
- [ ] Images have alt text.
- [ ] No color-only meaning.
- [ ] Tables include headers.
- [ ] Instructions are understandable without screenshots.
- [ ] Language is plain and learner-friendly.
